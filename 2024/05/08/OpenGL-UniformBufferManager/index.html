OpenGL - UniformBufferManager

本文最后更新于：2024年5月9日 凌晨

需求的函数功能

  • unsigned int UniformBufferManager::createUniformBuffer(GLsizeiptr size, GLenum usage);

    参数接受一个 GLsizeiptr 型变量 size，与一个 GLenum 型变量 usage，函数将创建一个大小为 size 个字节的，内存使用类型为 usage 的 Uniform Buffer 对象，并返回该对象的 ID。

  • void UniformBufferManager::uniformBufferBinding(unsigned int ubo, unsigned index);

    参数接受一个 Uniform Buffer 对象 ubo，与一个无符号整型变量 index 作为绑定点的下标。函数会将传入的 ubo 绑定到 index 绑定点。

  • void Shader::uniformBlockBinding(const std::string& name, unsigned int index);

    参数接受一个string 型变量 name，与一个无符号整型变量 index 作为绑定点的下标。参数将 Shader 中名为 name 的 Uniform Block 绑定到 index 绑定点。

  • void UniformBufferManager::setUniformBufferxxx(unsigned int ubo, const std::string& name, xxx value);

    参数接受一个 Uniform Buffer 对象 ubo，一个 string 型变量 name，与一个对应的 xxx 类型的变量（可以是数组）。函数将检查变量名为 name 的变量是否已经存在 ubo 中，如果是，将对应的内存中的数据修改为新的数据 value，否则使用 std140 布局，在未使用的内存中将数据 value 保存。

实现

实现上述函数，需要使用 map 数据结构，保证可以保存，以及快速查找当前 Uniform Buffer 内的存储情况。而 std140 布局，网上能找到很多关于布局规则的说明（有点绕），踩坑的地方会在注释中写出来。

  • uniformStruct 结构体

    uniformStruct 的成员变量保存了对应变量的基准对齐量，以及在 Buffer 中的偏移量。

  • 成员变量

    • std::map<std::string, uniformStruct>* uniformBufferStruct;

      变量名为键，变量的 uniformStruct 为键值的一个 map，保存了所有变量在 Buffer 中的信息，可以用变量名快速查找变量在 Buffer 中的信息。

    • std::map<unsigned int, unsigned int>* curUniformBufferIndex;

      Uniform Buffer 的 ID 为键，键值保存的是 ID 对应的 Uniform Buffer 内存的使用情况，指向当前未使用内存的前端（遵守 std140 规则）。

注意

  • 添加到 UniformBufferManager 中的变量名不能重复；
  • 添加变量的顺序需要和 Uniform Block 中的定义顺序相同；
  • 添加的变量名可以与 Uniform Block 中的变量名不同，只要类型以及顺序相同即可；
  • 不要添加超过创建 Buffer 内存大小的变量；
