OpenGL - Advanced Data & Advanced GLSL

本文最后更新于：2024年5月6日 晚上

1. 高级数据

对于已分配内存的的缓冲，填充缓冲的特定区域有两种方法。

函数接受一个缓冲目标、一个偏移量、数据的大小和数据本身作为它的参数。

通过调用 glMapBuffer 函数，OpenGL会返回当前绑定缓冲的内存指针。

1.1 分批顶点属性

使用 glBufferSubData 可以采取一种新的顶点属性布局方式。从原本的 123123123 式交错布局改为 111222333 式的分批布局。

使用两种不同的方式不会有什么大的变化。

1.2 复制缓冲

使用 glCopyBufferSubData 可以相对容易地从一个缓冲中复制数据到另一个缓冲中。

  • readtarget 和 writetarget：复制源和复制目标的缓冲目标。
  • readoffset 和 writeoffset：复制缓冲时使用的偏移量。
  • size：复制缓冲的区块大小。

如果复制操作的读写目标是同一缓冲目标类型，可以使用 GL_COPY_READ_BUFFER 和 GL_COPY_WRITE_BUFFER 作为临时的缓冲目标，并将这两个目标作为readtarget和writetarget参数。

2. 高级GLSL

内建变量 (Built-in Variable) 和 Uniform 缓冲对象 (Uniform Buffer Object)。

2.1 顶点着色器变量

  • gl_PointSize ：float 型变量，可以使用它来设置点的宽高（像素）。

    在顶点着色器中修改点大小的功能默认是禁用的，需要手动启用。

  • gl_VertexID：整型变量，储存了正在绘制顶点的当前 ID。当进行索引渲染的时候，这个变量会存储正在绘制顶点的当前索引。当不使用索引进行绘制的时候，这个变量会储存从渲染调用开始的已处理顶点数量。

2.2 片段着色器变量

  • gl_FragCoord：片段坐标，z 分量保存了片段的深度值。

    也可以使用 x 与 y 分量实现一些有趣的效果。

  • gl_FrontFacing：bool 型变量，如果当前片段是正向面的一部分那么就是true，否则就是false。

  • gl_FragDepth：float 型输出变量，可以在片段着色器内设置片段的深度值。

    通过 gl_FragDepth 设置深度值会导致 OpenGL 禁用所有的提前深度测试（无法在片段着色器允许之前得知所有片段的深度值），会造成一定的性能影响。

    • 值得一提的是，从OpenGL 4.2起，可以在片段着色器的顶部使用深度条件 (Depth Condition) 重新声明 gl_FragDepth 变量：

      condition可以为下面的值：

      条件 描述
      any 默认值。提前深度测试是禁用的，你会损失很多性能
      greater 你只能让深度值比gl_FragCoord.z更大
      less 你只能让深度值比gl_FragCoord.z更小
      unchanged 如果你要写入gl_FragDepth，你将只能写入gl_FragCoord.z的值

2.3 接口块

GLSL 提供了接口块 (Interface Block)，方便组织和管理各个着色器间变量数据的传递。与普通的输入输出变量相同的是，两个着色器中定义的块名应该保持一致。

2.4 Uniform 缓冲对象

Uniform 缓冲对象可以定义一系列在多个着色器程序中相同的全局Uniform变量。当使用 Uniform 缓冲对象的时候，我们只需要设置相关的 uniform 一次。

这个 Uniform 块使用最常用的 std140 布局，std140布局声明了每个变量的偏移量都是由一系列规则所决定的，这显式地声明了每个变量类型的内存布局，我们可以手动计算每个变量的偏移量。

std140 布局中最小的内存单位为 4 个字节，常见的规则如下：

类型 布局规则
标量 每个标量的基准对齐量为N。
向量 2N或者4N。这意味着vec3的基准对齐量为4N。
标量或向量数组 每个元素的基准对齐量与vec4的相同。
矩阵 储存为列向量的数组，每个向量的基准对齐量与vec4的相同。
结构体 等于所有元素根据规则计算后的大小，但会填充到vec4大小的倍数。
