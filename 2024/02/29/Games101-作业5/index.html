总览

(来自课程作业介绍PDF)

在这部分的课程中，我们将专注于使用光线追踪来渲染图像。在光线追踪中 最重要的操作之一就是找到光线与物体的交点。一旦找到光线与物体的交点，就 可以执行着色并返回像素颜色。在这次作业中，我们需要实现两个部分：光线的 生成和光线与三角的相交。本次代码框架的工作流程为：

  1. 从 main 函数开始。我们定义场景的参数，添加物体（球体或三角形）到场景 中，并设置其材质，然后将光源添加到场景中。
  2. 调用 Render(scene) 函数。在遍历所有像素的循环里，生成对应的光线并将 返回的颜色保存在帧缓冲区（framebuffer）中。在渲染过程结束后，帧缓冲 区中的信息将被保存为图像。
  3. 在生成像素对应的光线后，我们调用 CastRay 函数，该函数调用 trace 来 查询光线与场景中最近的对象的交点。
  4. 然后，我们在此交点执行着色。我们设置了三种不同的着色情况，并且已经 为你提供了代码。 你需要修改的函数是： • Renderer.cpp 中的 Render()：这里你需要为每个像素生成一条对应的光 线，然后调用函数 castRay() 来得到颜色，最后将颜色存储在帧缓冲区的相 应像素中。 • Triangle.hpp 中的 rayTriangleIntersect(): v0, v1, v2 是三角形的三个 顶点，orig 是光线的起点，dir 是光线单位化的方向向量。tnear, u, v 是你需 要使用我们课上推导的 Moller-Trumbore 算法来更新的参数。

编写

作业的介绍文档已经把新的框架以及作业内容介绍的很清楚了。

先来写Renderer.cpp 中的 Render()方法，该方法的主要目的是遍历所有像素，找到从观察点出发到某个像素的一个方向向量。此处仅介绍方法，详细推荐阅读Generating Camera Rays with Ray-Tracing.

  • NDC space：将像素坐标压缩到[0,1]内：

\[ \begin{gather*} PixelNDC_x = \dfrac{(Pixel_x + 0.5)}{ImageWidth}\\ PixelNDC_y = \dfrac{(Pixel_y + 0.5)}{ImageHeight} \end{gather*} \]

  • screen space：变换坐标轴位置

\[ \begin{gather*} PixelScreen_x = 2 * {PixelNDC_x} - 1\\ PixelScreen_y = 1 - 2 * {PixelNDC_y} \end{gather*} \]

  • x坐标乘以宽高比，恢复原始比例

  • x,y坐标都乘以可视角度的一半的tan值，便可以计算出其世界坐标。

Triangle.hpp 中的 rayTriangleIntersect()方法要实现课上讲过的MT算法，参数在总览中有介绍，MT算法可以看以前的笔记。
